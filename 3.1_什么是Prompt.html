3.1 什么是Prompt？

你对AI说的每句话都是Prompt — 输出质量90%取决于此
1
小白：Prompt这个词到处都能看到，它到底是什么？跟普通的"提问"有区别吗？
AI博士：Prompt就是你给AI模型的所有输入内容——问题、指令、背景信息、示例、约束条件的总和。它不只是一个"问题"，而是一整套沟通策略。
📌 关键认知：同一个模型（比如Claude 4.6），给它不同的Prompt，输出质量可以差10倍以上。Prompt质量是AI应用效果的第一决定因素，比模型选择更重要。
2
差 Prompt vs 好 Prompt
❌

差的Prompt

"帮我写个方案"

问题：
• 没说什么方案
• 没说给谁看
• 没说什么格式
• 没说多长
• 没有背景信息

结果：AI只能猜，
输出泛泛而谈

VS
✅

好的Prompt

"你是一位有10年经验的B2B SaaS产品经理。请为我们的项目管理工具写一份面向中小企业的推广方案。

要求：
1. 目标用户画像
2. 3个核心卖点
3. 推广渠道及预算分配
4. 用表格呈现，1000字左右"

结果：精准、结构化、可执行

3
Prompt 的构成要素
🎭

角色 (Role)

告诉AI扮演谁
"你是资深架构师"
为什么有效：激活模型中与该角色相关的知识模式

🎯

任务 (Task)

明确要做什么
"写一份技术调研报告"
为什么有效：让模型聚焦输出方向

📋

约束 (Constraint)

限定格式和范围
"用Markdown表格，800字内"
为什么有效：减少模型的输出空间

📎

上下文 (Context)

提供背景信息
"我们是一家20人的创业公司…"
为什么有效：让模型的回答贴合实际

📝

示例 (Examples)

给出参考样本
"参考以下格式：…"
为什么有效：比描述更高效地传达期望

🔧

输出格式 (Format)

指定输出结构
"以JSON/表格/分点返回"
为什么有效：程序可解析，减少后处理

4
Prompt 的三种层级
层级名称内容谁设定用户可见
1️⃣System PromptAI的底层人设和规则开发者❌ 不可见
2️⃣User Prompt用户的具体问题/指令用户✅ 可见
3️⃣Assistant PromptAI之前的回复(多轮对话)AI自己✅ 可见
AI博士：这三层组合在一起形成完整的Prompt。System Prompt是最关键的——它决定了AI的基本行为模式。比如Claude的System Prompt让它诚实、有帮助、无害；一个客服机器人的System Prompt会限制它只回答产品相关问题。
💡 Agent开发中：Prompt设计就是Harness的核心组件之一——CLAUDE.md / AGENTS.md 本质就是一份精心设计的System Prompt + 约束文档。
5

📝 本节要点

✅ Prompt不只是问题，而是角色+任务+约束+上下文+示例+格式的组合
✅ Prompt质量是AI输出效果的第一决定因素（>模型选择）
✅ 好Prompt的核心：具体、明确、有约束、给示例
✅ Prompt分三层：System Prompt(底层人设) + User Prompt(用户指令) + 对话历史
