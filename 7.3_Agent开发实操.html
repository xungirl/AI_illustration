🛠️ 推荐技术栈（2026）

模型层
• 首选：Claude API / GPT-4o API（效果最好）
• 私有部署：Qwen2.5-72B / DeepSeek-V3（成本可控）

Harness框架
• 编码Agent：Claude Agent SDK
• 复杂工作流：LangGraph
• 多Agent：CrewAI

RAG / 知识层
• Embedding：BGE-M3 / text-embedding-3
• 向量库：Chroma(轻量) / Milvus(生产)

工具连接
• MCP协议 + 自定义工具

可观测性
• LangSmith / LangFuse

部署
• Docker + 云服务(AWS/GCP/阿里云)
3

⚠️ 常见踩坑点

① 上来就搞多Agent：先用单Agent跑通，确认需要再拆分
② 忽略评估：没有自动化评测 = 在黑暗中优化
③ 过度依赖模型：该用规则的地方用规则，不是所有事都要LLM
④ 忽略错误处理：API超时、模型拒绝、工具失败——都要有降级方案
⑤ 不做成本估算：Token消耗可能比想象的多很多——先算账再开发

📝 本节要点

✅ Agent开发四步法：明确场景→选架构→开发迭代→评估上线
✅ 先从最小可用版本开始，Harness逐步完善
✅ 技术栈：模型API + Harness框架 + RAG + MCP + 可观测性
✅ 避免常见坑：过度设计、忽略评估、忽略成本
