MCP 协议

🔌 MCP = Model Context Protocol（模型上下文协议）

提出者：Anthropic（2024年底发布）

核心思想：给AI工具调用建立一个统一标准——就像USB是硬件设备的统一接口，MCP是AI工具的统一接口。

之前的问题：
• 每个AI平台的工具调用格式不同（OpenAI有OpenAI的，Claude有Claude的）
• 每个工具需要为每个平台单独适配
• 生态碎片化，工具复用困难

MCP解决的问题：
• 统一的工具描述格式（Tool Schema）
• 统一的调用协议（Request/Response）
• 统一的安全和权限模型
• 一次开发，所有支持MCP的AI都能用

MCP架构：
MCP Host（AI应用）↔ MCP Client（协议层）↔ MCP Server（工具提供方）

⚠️ 工具调用的安全原则

① 最小权限：Agent只能访问完成任务所需的最小工具集
② 人类审批：危险操作（删除、支付、发送）需人工确认
③ 沙箱隔离：代码执行在隔离环境中，不能访问主系统
④ 审计日志：记录所有工具调用，可追溯
