2
ReAct：最经典的Agent范式

🔄 ReAct = Reason(推理) + Act(行动)

用户："北京明天适合户外活动吗？"

Thought 1：我需要查北京明天的天气
Action 1：调用 get_weather("北京", "明天")
Observation 1：北京明天，晴，25°C，PM2.5: 35(优)

Thought 2：天气晴好，气温舒适，空气质量优，适合户外。让我推荐几个活动
Action 2：调用 search("北京户外活动推荐")
Observation 2：颐和园、奥森公园、长城……

Thought 3：信息足够了，可以给出完整建议
Final Answer："明天北京晴天25°C，空气质量优，非常适合户外活动！推荐：①颐和园 ②奥森公园 ③慕田峪长城……"
🔑 ReAct的关键：模型交替进行推理和行动，每步行动后观察结果再决定下一步。这比直接回答准确得多，因为它基于真实数据而非记忆。
3
常见Agent架构模式
模式原理优点适用场景
ReAct推理-行动-观察循环简单直观、通用性强通用任务，工具调用
Plan-and-Execute先规划完整方案再逐步执行全局视野、减少反复复杂多步任务
Reflection执行后自我反思和纠错提高质量、减少错误需要高准确率的场景
Tree of Thoughts探索多条推理路径选最优处理高不确定性问题数学、逻辑推理
LATS蒙特卡洛树搜索+反思在复杂空间中找最优解代码生成、策略规划

📝 本节要点

✅ Agent核心循环：感知→规划→执行→反思→循环
✅ ReAct是最经典的Agent范式：Reason+Act交替进行
✅ 常见架构：ReAct(通用)、Plan-and-Execute(复杂任务)、Reflection(高质量要求)
✅ 规划策略和反思深度是区分Agent质量的关键
