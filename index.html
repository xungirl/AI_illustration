1.1 什么是人工智能？

Artificial Intelligence — 从图灵测试到GPT的70年征程
1
小白：AI到底是什么？我感觉到处都在说AI，但每个人说的好像不一样？🤔
AI博士：好问题！AI（Artificial Intelligence，人工智能）是计算机科学的一个分支，目标是让机器能够执行通常需要人类智能才能完成的任务——比如理解语言、识别图像、做决策、学习新知识。
📌 学术定义（John McCarthy, 1956）："人工智能是制造智能机器的科学和工程，尤其是智能计算机程序。它与使用计算机理解人类智能的类似任务相关，但AI不必局限于生物学上可观察的方法。"

1.2 AI是怎么学习的？

Machine Learning — 从数据中自动发现规律的技术
1
小白：上节说AI通过数据学习，具体是怎么"学"的？跟人学习有什么区别？
AI博士：机器学习（Machine Learning）的核心思想：不是人写规则告诉机器怎么做，而是给机器大量数据，让它自己发现规律。这是AI从"编程时代"到"训练时代"的根本转变。

🐱 经典案例——猫狗识别：
① 收集10万张猫和狗的照片，每张标注"猫"或"狗"
② 将数据分为训练集(80%)和测试集(20%)
③ 模型在训练集上学习——第1轮可能只有50%准确率，第100轮达到95%
④ 在测试集上验证，确保模型不是"死记硬背"而是真的学会了
⑤ 部署后，给一张从没见过的照片也能准确判断猫或狗

三大学习范式
📚

监督学习

有老师教

模型学习输入→输出的映射

典型任务：分类、回归
例：垃圾邮件识别、房价预测
最成熟应用最广

自学成才
数据没有标签
模型自己发现隐藏结构

典型任务：聚类、降维
例：用户分群、异常检测

🎮

试错进步
通过奖励/惩罚信号学习
在环境中不断试验

例：AlphaGo、机器人控制

🆕 还有一种重要的范式叫自监督学习(Self-supervised Learning)——大模型就是用这种方式训练的！不需要人工标注，通过"遮住一个字让模型猜"来自动生成训练目标。这让AI可以利用互联网上海量的无标注文本来学习。

小白：深度学习和机器学习是什么关系？
AI博士：深度学习(Deep Learning)是机器学习的一个子集，使用多层神经网络来学习。"深"指的是网络层数多（几十到几百层）。它是当前AI最强大的技术！

🧠 神经网络怎么工作？（简化版）

灵感来源：模拟人脑神经元的连接方式

输入层（接收数据）→ 隐藏层1（提取基础特征）→ 隐藏层2（提取高级特征）→ … → 输出层（做出判断）

以图像识别为例：
第1层：识别边缘和线条
第2层：识别眼睛、耳朵等部件
第3层：组合部件识别整体形状
第4层：判断这是"猫"

每一层自动提取越来越抽象的特征——这就是"深度"的威力！
5
关键突破：Transformer
AI博士：2017年Google发表了一篇论文《Attention Is All You Need》，提出了Transformer架构——这改变了一切。

⚡ 为什么Transformer这么重要？

1️⃣ 注意力机制(Attention)：让模型能关注输入中最相关的部分，而不是逐字顺序处理
2️⃣ 并行计算：之前的RNN必须逐步处理，Transformer可以同时处理所有位置，训练速度飙升
3️⃣ 可扩展性：模型越大、数据越多，效果越好——这催生了"大力出奇迹"的大模型路线

🏗️ 基于Transformer构建的模型：
• GPT系列（Decoder-only）→ 文本生成、ChatGPT
• BERT（Encoder-only）→ 文本理解、搜索
• T5/BART（Encoder-Decoder）→ 翻译、摘要
6
模型怎么"学习"？
AI博士：不管多复杂的模型，学习过程的本质都一样——不断调整参数，让预测结果更接近正确答案。

🔄 训练循环（Training Loop）

① 前向传播：数据通过网络得到预测结果
② 计算损失：预测结果和正确答案对比，算出"错了多少"
③ 反向传播：计算每个参数对错误的"贡献"（梯度）
④ 更新参数：根据梯度调整参数，让下次预测更准
⑤ 重复：以上步骤重复数百万次，直到模型收敛

💡 类比：就像练投篮——投一球(前向)→看偏了多少(损失)→分析是力度还是角度不对(反向)→调整姿势(更新)→再投(重复)。几万次后就能百发百中！

小白：那过拟合和欠拟合是什么意思？经常听到这两个词。
📖

过拟合(Overfitting)

死记硬背型
训练数据上99%
新数据上60%
模型记住了答案
而不是学会了规律

解决：更多数据、正则化
Dropout、早停

VS
😴

欠拟合(Underfitting)

学渣型
训练数据上60%
新数据上55%
模型太简单
没学到有用的东西

解决：更复杂模型
更多特征、更长训练

AI博士：好的模型要找到恰好的平衡——既不死记硬背，也不马虎了事。这叫做泛化能力(Generalization)，是评价模型好坏的核心指标。

2.2 大模型怎么工作的？

预训练 → SFT → RLHF — 三步炼成超级学霸
1
小白：大模型是怎么从"一堆参数"变成ChatGPT这样能聊天的？
AI博士：分三个阶段，就像培养一个学生：先博览群书，再学会对话，最后学会做人。
2
第一步：预训练 Pre-training

📖 目标：学会语言规律和世界知识

方法：在互联网规模的文本上做"完形填空"
任务：给定前面的文字，预测下一个字（Next Token Prediction）

示例：
输入：「今天北京的天气非常」
模型预测：「好」(概率0.35)「晴」(0.28)「热」(0.15)...

关键数据：
• 训练数据：数万亿Token（整个互联网、书籍、代码、论文）
• 训练时长：数周~数月，消耗数千万美元算力
• 学到了什么：语法、语义、推理、世界知识、编程能力……

⚠️ 预训练后的模型很有"知识"，但不会"聊天"——它只会接话（补全文本），不会按指令回答问题。
3
第二步：SFT 监督微调

🎯 目标：学会按指令回答问题

方法：用人工编写的"问题-高质量回答"数据对来训练
数据量：通常只需要数万~数十万条高质量对话（远少于预训练）

示例训练数据：
问："请解释什么是光合作用"
答："光合作用是植物利用阳光将二氧化碳和水转化为葡萄糖和氧气的过程……"

SFT之后的变化：
• 从"接话"变成"回答" ✅
• 能遵循指令格式 ✅
• 回答更有结构性 ✅
• 但可能啰嗦/讨好/回避问题 ⚠️

🔑 数据质量极其重要：SFT数据的质量直接决定模型的对话风格和能力上限。"少而精"胜过"多而杂"。
4
第三步：RLHF 人类反馈强化学习

👨‍🏫 目标：学会什么回答"更好"

全称：Reinforcement Learning from Human Feedback

过程分两步：

Step 1 — 训练奖励模型(RM)：
• 对同一个问题，让模型生成多个回答
• 人类标注员对这些回答排序（A>B>C）
• 用排序数据训练一个"评分模型"

Step 2 — PPO强化学习优化：
• 模型生成回答 → 奖励模型打分 → 模型根据分数调整
• 高分回答的生成概率增加，低分回答的概率减少
• 重复数万次，模型越来越符合人类偏好

RLHF后的提升：
• 回答更有用、更安全、更诚实 ✅
• 减少有害/偏见输出 ✅
• 拒绝不合理请求 ✅
• 这一步让AI从"能说话"进化到"说人话" 🚀
5
Token：大模型的语言单位
小白：Token到底是什么？为什么大家都在说Token？
AI博士：大模型不是按"字"或"词"来处理文本的，而是按Token。Token是模型切分文本的最小单位。

🔤 Token速查

• 英文：1个Token ≈ 4个字母 ≈ 0.75个单词
• 中文：1个Token ≈ 1-2个汉字
• "Hello World" = 2 Tokens
• "人工智能" ≈ 2-4 Tokens（取决于分词器）

上下文窗口（Context Window）：
• GPT-4o：128K Token ≈ 一本300页的书
• Claude 3.5：200K Token ≈ 约500页
• Gemini 1.5：1M Token ≈ 约2500页

上下文窗口决定了模型能"记住"多少内容。超出窗口的早期内容会被遗忘——这就是为什么Agent需要记忆管理！
6
推理：逐字生成的秘密
AI博士：当你发出一个问题，大模型是这样生成回答的：

🔄 自回归生成过程

输入：「什么是机器学习？」

第1步 → 生成：「机」（概率最高的下一个Token）
第2步 → 生成：「器」（给定"机"后概率最高的）
第3步 → 生成：「学」
第4步 → 生成：「习」
第5步 → 生成：「是」
……逐Token生成，直到模型输出结束符号

关键细节：
• 不是随机选，而是按概率分布采样
• Temperature参数控制随机性：低=更确定，高=更创意
• Top-p参数控制候选范围
• 生成是单向的——已生成的字不会被修改
💡 这就解释了为什么AI有时会"一本正经地胡说"——它只是在做概率预测，不是在"查字典"。每个字都是基于前文最可能的延续，但概率高不等于事实正确。

Few-shot 详解

📝 Few-shot = 给AI看几个例子再让它做

Zero-shot（零样本）：直接问，不给例子
"判断这句话的情感：这个产品太差了" → 负面

One-shot（一个例子）：
"例：这部电影太精彩了 → 正面
请判断：这个产品太差了 → ?"

Few-shot（多个例子）：
"例1：味道很好吃 → 正面
例2：等了两小时 → 负面
例3：还行吧 → 中性
请判断：性价比超高的 → ?"

🔑 关键：例子要覆盖不同类型和边界情况，3-5个通常足够。例子的格式就是你期望的输出格式——AI会严格模仿。
3
CoT 思维链详解

🔗 CoT = 让AI"展示解题步骤"

不用CoT：
"一个商店有15个苹果，卖了8个，又进了12个，问还有多少？" → "19个"

用CoT：
"一个商店有15个苹果，卖了8个，又进了12个，问还有多少？请一步一步推理。"
→ "让我一步步分析：
第一步：初始有15个苹果
第二步：卖了8个，剩下15-8=7个
第三步：又进了12个，7+12=19个
所以还有19个苹果。"

🔑 为什么CoT有效？
1. 逼迫模型进行中间推理而不是直接跳到结论
2. 中间步骤可以被检查，发现逻辑错误
3. 复杂推理任务准确率可提升20-40%

📌 触发词："请一步一步思考"/"Let's think step by step"/"先分析再给结论"
4
结构化输出详解
AI博士：当AI的输出需要被程序处理时，结构化输出是必须的。

📊 常用结构化格式

JSON格式：（最常用，API友好）
"分析以下评论的情感和关键词，以JSON返回：
{"sentiment": "正面/负面/中性", "keywords": [...], "confidence": 0.0-1.0}"

Markdown表格：（适合人工阅读）
"用表格对比方案A和B：| 维度 | 方案A | 方案B |"

YAML/XML：（配置文件、文档结构）
"以YAML格式输出API接口定义"

🔑 技巧：在Prompt中直接给出输出的schema（模板），AI会严格遵循。比直接说"用JSON返回"效果好得多。
5
高级技巧组合

🚀 实战组合技

① CoT + Few-shot：给几个带推理过程的示例，AI会模仿同样的推理深度

② 角色 + CoT + 结构化："你是资深审计师，请逐条审查以下合同条款，用表格列出：条款编号/风险等级/问题描述/修改建议。请先分析再给结论。"

③ Self-Consistency：让AI生成3次，取共识答案——适合数学推理等有唯一正确答案的场景

④ ReAct模式："思考→行动→观察→再思考"——这就是Agent的核心Prompt模式（第5章详讲）

📖 RAG = Retrieval Augmented Generation（检索增强生成）

核心思想极其简单：AI回答问题之前，先去知识库搜索相关资料，然后基于资料回答。

类比：
• 没有RAG = 闭卷考试（全靠记忆，容易编）
• 有了RAG = 开卷考试（可以翻书，答案有据可查）

这个思想虽然简单，却是目前解决大模型幻觉和知识时效性问题最有效、最主流的方案！

4.2 RAG的工作原理

索引 → 检索 → 增强 → 生成 — 完整工作流拆解
1
AI博士：RAG系统分为两个阶段：离线索引（提前准备知识库）和在线查询（实时回答问题）。

🏗️ 阶段一：离线索引（Indexing）—— 建图书馆

1️⃣ 文档加载：导入PDF、Word、网页、数据库等各种格式
2️⃣ 文档切片：把长文档切成小段（Chunk），通常200-1000字
3️⃣ Embedding：把每个Chunk转成向量（一组数字）
4️⃣ 存入向量数据库：Chunk文本+向量一起存储和索引

这个过程只需做一次（或文档更新时增量更新），相当于建好一个可以语义搜索的图书馆。
2

🔍 阶段二：在线查询（Querying）—— 去图书馆查资料

1️⃣ 用户提问："公司的年假政策是什么？"
2️⃣ Query Embedding：把问题也转成向量
3️⃣ 向量检索：在向量数据库中找到最相似的Top-K个Chunk
4️⃣ Prompt组装：把检索到的Chunk拼到Prompt里
   "根据以下参考资料回答用户问题： [Chunk1] [Chunk2] [Chunk3] 用户问题：公司的年假政策是什么？"
5️⃣ LLM生成：大模型基于参考资料生成回答
6️⃣ 返回结果：回答+引用来源（可溯源！）

4.3 向量与Embedding

把文字变成数字 — 语义搜索的数学基础
1
小白：Embedding到底是什么？为什么把文字变成数字就能搜索语义了？
AI博士：Embedding就是用一组数字（向量）来表示一段文字的"含义"。关键特性：含义相近的文字，向量也相近。

🎯 直观理解

想象一个多维空间（比如1536维），每段文字是空间中的一个点：

• "猫" 和 "狗" 的点距离很近（都是宠物）
• "猫" 和 "汽车" 的点距离很远（完全不同的概念）
• "苹果手机" 和 "iPhone" 几乎重叠（同一个东西）
• "苹果手机" 和 "苹果水果" 距离较远（虽然同名但含义不同）

搜索时：把查询也变成一个点，然后找空间中离它最近的那些点——这就是语义搜索！

相似度计算

📐 三种常用相似度度量

余弦相似度（Cosine Similarity）：最常用
衡量两个向量方向的相似性，值域[-1, 1]，1=完全相同
不受向量长度影响，只看方向

欧氏距离（L2 Distance）：
两点之间的直线距离，值越小越相似
对向量幅度敏感

点积（Dot Product）：
同时考虑方向和幅度
计算最快，适合归一化后的向量

🔑 大多数场景用余弦相似度即可。如果embedding已经归一化，三种方法结果等价。

切片策略

Reranking 重排序

🏆 Reranking = 检索后的"二次面试"

问题：向量检索的Top-20结果中，真正最相关的可能不在最前面

方案：用一个更强的模型（Cross-Encoder）对检索结果精排

工作流：
1. 向量检索召回Top-20（粗筛，速度快）
2. Reranker对20条逐一打分（精排，效果好）
3. 取精排后的Top-5送给LLM

常用Reranker：
• Cohere Rerank（API服务，效果好）
• BGE-Reranker（开源，中文友好）
• Jina Reranker v2

🔑 Reranking可以将检索相关性提升20-30%，是投入产出比最高的优化手段之一。

评估指标

📊 RAG系统的核心评估维度

检索质量：
• 召回率(Recall@K)：相关文档有没有被找到？
• 精确率(Precision@K)：找到的是不是真相关？
• MRR(Mean Reciprocal Rank)：相关文档排在第几？

生成质量：
• 忠实度(Faithfulness)：回答是否忠于检索到的资料？
• 答案相关性：回答是否真正解答了用户的问题？
• 有害拒绝率：有资料时是否仍拒绝回答？

🔧 评估工具：RAGAS、TruLens、LangSmith都提供RAG自动评估。

MCP 协议

🔌 MCP = Model Context Protocol（模型上下文协议）

提出者：Anthropic（2024年底发布）

核心思想：给AI工具调用建立一个统一标准——就像USB是硬件设备的统一接口，MCP是AI工具的统一接口。

之前的问题：
• 每个AI平台的工具调用格式不同（OpenAI有OpenAI的，Claude有Claude的）
• 每个工具需要为每个平台单独适配
• 生态碎片化，工具复用困难

MCP解决的问题：
• 统一的工具描述格式（Tool Schema）
• 统一的调用协议（Request/Response）
• 统一的安全和权限模型
• 一次开发，所有支持MCP的AI都能用

MCP架构：
MCP Host（AI应用）↔ MCP Client（协议层）↔ MCP Server（工具提供方）

🧮 为什么Agent需要Harness——可靠性的数学

假设Agent每一步成功率95%（听起来很高），但：

• 5步任务：0.95⁵ = 77% 成功率
• 10步任务：0.95¹⁰ = 60% 成功率
• 20步任务：0.95²⁰ = 36% 成功率
• 50步任务：0.95⁵⁰ = 8% 成功率

这就是为什么Agent Demo看起来很好（3-5步），但到生产环境（20-50步）就崩溃了。

Harness的作用就是在每一步加入验证和恢复机制——把单步成功率从95%提升到99%，或者在失败时自动重试/回退。这就是可靠性工程的本质。

7.4 安全、伦理与边界

能力越大责任越大 — 负责任的AI
1
🌍 🤖 🔒 ⚖️

⚠️ AI系统的核心风险

🤥 幻觉与错误：AI可能生成看似正确的假信息，在关键决策中造成严重后果
⚖️ 数据偏见：训练数据的偏见会被模型放大——性别、种族、地域偏见
🔒 隐私泄露：敏感数据可能被模型记忆并在其他对话中泄露
🎭 深度伪造：文字/图片/视频/语音都可以被AI高度仿真
🤖 Agent风险：Agent能执行真实操作——错误操作可能造成不可逆损害
💰 过度依赖：长期依赖AI可能导致人类技能退化和判断力下降
2

✅ 负责任地使用AI — 6条原则

1️⃣ 人工验证：AI输出的重要信息必须人工验证，尤其是数据、事实、法律条款
2️⃣ 数据脱敏：发送给AI的数据要去除个人隐私（姓名、身份证、银行卡）
3️⃣ 可解释性：AI的决策过程应该可以被理解和审查
4️⃣ 人类决策权：最终决策权始终在人类手中，AI是建议而非命令
5️⃣ 持续监控：部署后持续监控AI行为，发现偏移及时纠正
6️⃣ 透明度：让用户知道他们在和AI交互，不要伪装成人类
